The game of boxes

Level 1
  • Static Positioning is the default position strategy
  • No change observed when using position: static
  • Positioning attributes like (left/right/top/bottom) don't work.

Level 2
  • Absolute Positioning
  • Absolute elements break the flow
  • Positioning attributes like (left/right/top/bottom) work.
  • If parent also absolute, then absolute children become relative to parent, but parent breaks the flow of the page.
  • If no position attribute in parent then, absolute elements are placed w.r.t. the page.

Other thing to note is the difference between 50% and 50vw

Level 3
  • Relative Positioning
  • Relative elements don't break the flow
  • Relative means wrt to original position (according to the flow)
  • Relative elements appear above static elements, because of in-built z-index property
  • If parent -> relative but child -> absolute, then, children are placed "absolutely" wrt to the parent. Parent does not break flow.

Other thing to note is the difference between 50% and 50vw
